# Project Rules & Operational Protocols

> [!IMPORTANT]
> These rules must be followed strictly for every task. No unauthorized changes or deviations are permitted.

## 1. Structural Integrity & Scope
- **Strict Scope**: Do not touch or modify any files except those explicitly necessary for the current task.
- **No Refactoring**: Avoid unsolicited code cleanup or refactors unless requested.
- **Rule Ledger**: Always consult this file before starting any work.

## 2. UI & Design System
- **Preserve Theme**: Never change the system's color scheme or design tokens without explicit instruction.
- **Visual Consistency**: Any new UI elements must perfectly match existing headers, fonts, and color palettes.
- **Follow Screenshot Direction**: Use provided screenshots as the primary source of truth for UI discrepancies.

## 3. Communication & Execution
- **Listen to User**: Follow user instructions strictly. If the user says "do it this way," do not suggest alternatives unless asked.
- **Artifacts**: Maintain `task.md`, `implementation_plan.md`, and `walkthrough.md` for transparency.
